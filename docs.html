Getting Started › Quick Start

Quick Start

Get your first job live and accepting applications in under 10 minutes. No technical setup required for the basics.

Before you begin

You will need a Hire Flow account. If you do not have one yet, start your free trial — no credit card required.

💡
Tip: The free Starter plan lets you post up to 3 roles and explore the full pipeline before committing to a paid plan.

Step by step

  • 01

    Create your workspace

    Sign up and set your company name, logo and timezone. This takes about 2 minutes and sets up your default pipeline template.

  • 02

    Post your first job

    Go to Jobs → New Job. Fill in the role title, department, location and description. Hire Flow generates a public careers page URL automatically.

  • 03

    Customise your pipeline stages

    The default pipeline includes Applied, Screening, Interview, and Offer. You can add, rename or remove stages to match your process.

  • 04

    Invite your team

    Go to Settings → Team and invite hiring managers or interviewers. Assign them to specific roles to control what they can see and do.

  • 05

    Share your careers page

    Copy your public careers URL from Settings → Careers Page and add it to your website. Applications will flow in automatically.

Getting Started › Installation

Installation & Setup

Install the Hire Flow SDK or embed the careers widget directly into your existing website or app.

JavaScript SDK

Install via npm or yarn:

# npm
npm install @hireflow/sdk

# yarn
yarn add @hireflow/sdk

Initialise the client

import { HireFlow } from '@hireflow/sdk'

const client = new HireFlow({
  apiKey: 'hf_live_xxxxxxxxxxxx',
  workspace: 'your-workspace-id'
})
⚠
Never expose your API key in client-side code. Always initialise the SDK on the server side and use scoped tokens for any browser-facing calls.

Careers page embed

You can embed the Hire Flow careers widget directly into any webpage with a single script tag:

<div id="hireflow-jobs"></div>
<script
  src="https://cdn.hireflow.io/embed.js"
  data-workspace="your-workspace-id"
  data-theme="dark"
></script>
Getting Started › Authentication

Authentication

All API requests must be authenticated using an API key passed in the request header.

API Keys

Generate API keys from your workspace under Settings → API. Each key has a type:

Key TypePrefixUse case
Livehf_live_Production environment
Testhf_test_Development and testing
Scopedhf_scope_Read-only browser access

Sending the key

curl https://api.hireflow.io/v1/jobs \
  -H "Authorization: Bearer hf_live_xxxxxxxxxxxx" \
  -H "Content-Type: application/json"
Core Concepts › Pipeline

The Pipeline

A pipeline is the sequence of stages a candidate moves through for a given role. Each job has its own pipeline and you can customise it freely.

Default stages

StageDescription
AppliedCandidate has submitted their application
ScreeningInitial review — resume screen or async video
InterviewOne or more live interview rounds
OfferOffer sent, awaiting acceptance
HiredOffer accepted, candidate archived as hired

Custom stages

You can add any number of custom stages between the defaults. Common additions include Take-home Assessment, Reference Check, and Background Verification.

💡
Pipeline templates let you save a stage configuration and reuse it across multiple roles. Go to Settings → Pipeline Templates to manage them.
Core Concepts › Candidates

Candidates

Every person who applies to a role becomes a candidate record in Hire Flow. Candidate profiles centralise all their information, activity, and feedback in one place.

Candidate profile

A candidate profile contains:

  • — Personal details and contact info
  • — Resume and application materials
  • — Stage history and timestamps
  • — Scorecard submissions from interviewers
  • — Internal notes and @mentions
  • — Email communication log

Bulk actions

Select multiple candidates in the pipeline view to move them to a new stage, send a bulk email, or archive them all at once. Bulk actions respect individual email preferences and unsubscribe statuses.

Core Concepts › Scorecards

Scorecards

Scorecards give every interviewer a structured way to evaluate candidates consistently. They eliminate vague gut-feel feedback and create a record that the whole team can reference.

Creating a scorecard

  • 01

    Go to the role settings

    Open the job, click Settings, then Scorecards.

  • 02

    Add attributes

    Define the skills or traits you want evaluated — e.g. Communication, Technical Skills, Culture Add.

  • 03

    Set the rating scale

    Choose from 1–5 stars, Yes/No, or a custom label scale.

  • 04

    Assign to stages

    Attach the scorecard to the relevant interview stage so it appears automatically after each interview.

Core Concepts › Stages

Pipeline Stages

Stages define the checkpoints in your hiring process. Each stage can have its own automations, scorecards, email templates, and assigned reviewers.

Stage automations

When a candidate moves into a stage, Hire Flow can automatically send an email, schedule a reminder, or trigger a webhook. Configure automations in the stage settings panel.

💡
Example automation: When a candidate reaches the Offer stage, automatically send the offer letter template and notify the hiring manager in Slack.

API Reference › Candidates API

Candidates API

Create, read, update and move candidates through your pipeline programmatically.

Endpoints

GET /candidates

List all candidates across all jobs. Supports filtering by stage, job, date range and search query.

POST /candidates

Create a new candidate record and optionally add them directly to a specific job pipeline.

{
  "first_name": "Amara",
  "last_name": "Dube",
  "email": "amara@example.com",
  "job_id": "job_abc123",
  "stage": "screening"
}

POST /candidates/:id/move

Move a candidate to a different pipeline stage.

DELETE /candidates/:id

Archive a candidate. This is a soft delete — the record is retained for compliance purposes.

API Reference › Jobs API

Jobs API

Manage job postings, retrieve open roles, and control their visibility programmatically.

Endpoints

GET /jobs

Returns all jobs in your workspace. Filter by status (open, closed, draft), department, or location.

POST /jobs

Create a new job posting. Set it as a draft until ready to publish.

{
  "title": "Senior Product Designer",
  "department": "Design",
  "location": "Remote",
  "status": "open",
  "description": "..."
}

POST /jobs/:id/close

Close a job and stop accepting new applications. Existing candidates in the pipeline are unaffected.

API Reference › Webhooks

Webhooks

Webhooks allow Hire Flow to notify your systems in real time when events occur — like a candidate moving stages or an offer being accepted.

Available events

EventDescription
candidate.createdNew candidate added to a pipeline
candidate.stage_changedCandidate moved to a new stage
offer.sentOffer letter sent to a candidate
offer.acceptedCandidate accepted an offer
job.closedA job was closed

Verifying payloads

All webhook payloads include an X-HireFlow-Signature header. Verify it using your webhook secret to ensure the request is genuine.

const signature = req.headers['x-hireflow-signature']
const expected = crypto
  .createHmac('sha256', process.env.WEBHOOK_SECRET)
  .update(req.rawBody)
  .digest('hex')

if (signature !== expected) {
  return res.status(401).send('Invalid signature')
}
Integrations › Slack

Slack Integration

Connect Hire Flow to your Slack workspace to receive real-time hiring notifications, submit scorecard feedback, and keep your team aligned without leaving Slack.

Setup

  • 01

    Go to Integrations

    In your Hire Flow workspace, navigate to Settings → Integrations → Slack.

  • 02

    Authorise the Slack app

    Click Connect and authorise Hire Flow to access your Slack workspace. You will need Slack admin permissions.

  • 03

    Choose your notification channel

    Select the channel where hiring updates should be posted. You can set different channels per job or department.

What gets sent to Slack

EventNotification
New applicationPosted to the role's designated channel
Stage changeNotifies the assigned hiring manager
Scorecard submittedSummary posted to the hiring team channel
Offer acceptedCelebration message to the team channel 🎉
Integrations › Calendar Sync

Calendar Sync

Hire Flow integrates with Google Calendar and Outlook to make scheduling interviews frictionless for both your team and candidates.

Supported calendars

Google Calendar and Microsoft Outlook / Office 365 are both fully supported. Personal calendar accounts can be connected per team member from their profile settings.

Self-serve scheduling

Candidates receive a scheduling link where they can pick from available slots based on your interviewers' real calendars. No back and forth required. Once booked, both sides receive calendar invites automatically.

💡
Buffer times can be set per interviewer to prevent back-to-back interviews. Configure this in Settings → Calendar → Buffer.

Integrations › ATS Migration

ATS Migration

Moving from Greenhouse, Lever, or another ATS? Hire Flow provides a guided migration tool that imports your historical data with no data loss.

What gets migrated

  • — All historical candidate records
  • — Job postings (open and closed)
  • — Interview notes and scorecard history
  • — Team member accounts and permissions
  • — Custom fields and pipeline configurations
💡
Enterprise customers get a dedicated migration specialist to manage the process end to end. Contact your account manager to get started.
Account › Team Management

Team Management

Invite your hiring team, assign roles and control who can see and do what across your workspace.

Inviting members

Go to Settings → Team → Invite Member. Enter their email address and choose their role. They will receive an invitation email with a link to set up their account.

Roles

RoleWhat they can do
AdminFull access including billing and settings
RecruiterManage all jobs, candidates and pipelines
Hiring ManagerView and act on assigned roles only
InterviewerView assigned candidates and submit scorecards
Account › Permissions

Permissions

Fine-grained permissions let you control exactly what each team member can access, down to individual jobs and candidate records.

Job-level permissions

When creating or editing a job, you can specify which team members have access to that role's pipeline. Members not listed will not see the job or its candidates in their dashboard.

⚠
Note: Admins always have full access to all jobs regardless of job-level permissions. This cannot be changed.
